>	Используя File API, добавленный к DOM в HTML5, в веб-приложениях теперь можно запрашивать пользователя выбрать локальные файлы и затем читать содержимое этих файлов. Выбор файлов может осуществляться с помощью элемента [`<input>`](https://developer.mozilla.org/ru/docs/Web/HTML/Reference/Elements/input) или drag and drop.

>Если вы хотите использовать DOM File API в расширениях или коде Chrome, используйте. На самом деле, в таком случае вам необходимо ознакомиться с дополнительными нюансами. См. статью Using the DOM File API in chrome code для подробностей.

# Доступ к выбранным файлам

> [!info]
> >Рассмотрим следующий код:
> ```
> ><input type="file" id="input" multiple />
> 
> ```

>File API делает возможным доступ к [`FileList`](https://developer.mozilla.org/ru/docs/Web/API/FileList), который содержит объекты [`File`](https://developer.mozilla.org/ru/docs/Web/API/File), которым соответствуют файлы, выбранные пользователем.

>Атрибут `multiple` элемента `input` позволяет пользователю выбрать несколько файлов.

> [!info]
> Обращение к одному выбранному файлу с использованием классической DOM-модели:
> ```
> const selectedFile = document.getElementById("input").files[0];
> ```



> [!info]
> >Обращение к одному выбранному файлу через JQuery
> ```
> var selectedFile = $("#input").get(0).files[0];
> 
> var selectedFile = $("#input")[0].files[0];
> ```

> [!info]
> **Примечание:** Ошибка "files is undefined" означает что был выбран не один HTML-элемент, а список элементов, возвращаемый jQuery. Необходимо уточнить, у какого именно элемента требуется вызвать метод "files"

# Доступ к выбранным файлам через событие change

>Также возможно (но не обязательно) получить доступ к [`FileList`](https://developer.mozilla.org/ru/docs/Web/API/FileList) через событие `change`. Нужно использовать [`EventTarget.addEventListener()`](https://developer.mozilla.org/ru/docs/Web/API/EventTarget/addEventListener) чтобы добавить обработчик события `change`, как показано здесь:

> [!NOTE]
> ```
> const inputElement = document.getElementById("input");
> 
> inputElement.addEventListener("change", handleFiles, false);
> 
> function handleFiles() {
>   const fileList = this.files;
> }
> ```

# Получение информации о выделенных файлах 

Объект [`FileList`](https://developer.mozilla.org/ru/docs/Web/API/FileList) предоставляемый классическим DOM содержит все файлы выбранные пользователем, каждый из которых представляет собой объект [`File`](https://developer.mozilla.org/ru/docs/Web/API/File). Вы можете определить сколько файлов выбрал пользователь проверяя значение атрибута длины (`length`) списка файлов:

> [!NOTE]
> ```
> var numFiles = files.length;
> ```

> Всего существует три атрибута, предоставляемых объектом [`File`](https://developer.mozilla.org/ru/docs/Web/API/File), которые содержат полезную информацию о файле.
- ### Name : Содержит имя файла, доступна только для чтения
- ### Size : Размер файла в байтах
- ### Type : MIME тип файла, или пустая строка (`""`) если тип файла не может быть определён

# Использование метода click() скрытых элементов выбора файла

> Начиная с Gecko 2.0, вы можете скрыть непривлекательный элемент [`<input>`](https://developer.mozilla.org/ru/docs/Web/HTML/Reference/Elements/input) и предоставить свой собственный интерфейс для открытия диалогового окна выбора и отображения файла или файлов, выбранных пользователем. Вы можете сделать это, присвоив свойству display элемента input значение none (display:none) и вызывая метод `click()` скрытого элемента [`<input>`](https://developer.mozilla.org/ru/docs/Web/HTML/Reference/Elements/input).

#### Рассмотрим следующую разметку HTML:

> [!NOTE]
> ```
> <input
>   type="file"
>   id="fileElem"
>   multiple
>   accept="image/*"
>   style="display:none"
>   onchange="handleFiles(this.files)" />
> <a href="#" id="fileSelect">Select some files</a>
> ```
