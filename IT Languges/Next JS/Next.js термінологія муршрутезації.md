
> Next.js використовує файлову систему, як основу маршрутезації

### Структура папок

`app/` directory — основна папака для App router ( Next.js 13+ ) кожна папка в середині відповідаж сегменту маршруту.
`pages/` directory — старий підхід ( Pages router ), **Досі підтримується**

### Сегменти маршруту

> Кожна папка це сегмент маршруту, якщо до файла з сторінкою веде 2 папки, наш путь буде пролягати через ці 2 назви папок

### Спеціальні файли

| Файл          | Призначення                                                               |
| ------------- | ------------------------------------------------------------------------- |
| page.tsx      | Унікальний UI сторінки, робить маршрут публічним                          |
| layout.tsx    | UI для дочірніх компонентів, до яких він прив'язаний                      |
| template.tsx  | Той самий layout тільки більш гнучкіший, може змінюватись під час рендеру |
| loading.tsx   | UI завантаження (обгортає в `<Suspense>`)                                 |
| error.tsx     | UI помилки (обгортає в `<ErrorBoundary>`)                                 |
| not-found.tsx | UI для 404                                                                |
| route.ts      | API endpoint (замість `pages/api/`)                                       |

### Типи маршрутів

**Статичні** — звичайні папки:

```
app/about/page.tsx  →  /about
```

**Динамічні** — папка у квадратних дужках

```
app/blog/[slug]/page.tsx  →  /blog/anything
```

**Catch-all** — Перехоплює кілька сегментів

```
app/docs/[...slug]/page.tsx  →  /docs/a/b/c
```

**Optional catch-all** — Подвійна дужки

```
app/docs/[[...slug]]/page.tsx  →  /docs  або  /docs/a/b
```

### Групи та спеціальні папки

**Route group** ==(назва папки)== — групує маршрути без впливу на url

```
app/(marketing)/about/page.tsx  →  /about
app/(shop)/products/page.tsx    →  /products
```

**Private Folders** ==(_назва)== — Робить папку приватною, її не буде видно для маршрутизатора
**Parallel Routes** ==(@назва)== — Папка для паралельної маршрутизації
**Intercepring Routes** ==(.), (..), (...)== — перехоплює навігацію для показу контенту в модалці, не покидаючи поточну сторінку

### Навігація

- <Link/> — клієнтська навігація, не перезавантажує сторінку
- useRouter() — программа навігації (`router.push('/path')`)
- redirect() — Серверний редірект
- usePathname() — поточний шлях
- useSearchParams() — query параметри (`?foo=bar`)
- useParams — динамічні параметри (`[slug]`)

### Рендеринг стратегії

- **SSR** (Server-Side Renderign) — рендер на сервері при кожному запиті
- **SSG** (Static Site Generation) — рендер під час білду
- **ISR** (Incremental Static Regeneration) — статика з фоном оновлення (`revalidate`)
- **CSR** (Client-Side Rendering) — `"use client"` компоненти
